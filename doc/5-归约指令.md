# 5. 归约指令

本组包含以下指令：

- `tredmin`
- `tredminRow`
- `tredminCol`
- `tredmax`
- `tredmaxRow`
- `tredmaxCol`
- `tredsum`
- `tredsumRow`
- `tredsumCol`
- `tredavg`
- `tredavgRow`
- `tredavgCol`
- `tpol2max`
- `tpol2avg`

## 1. 通用规则

- 输出 `trd` 只能是普通寄存器
- 输入 `trs1` 只能是普通寄存器
- `trs2z` 是输出底板寄存器，可为普通寄存器或 `0` 寄存器
- 当 `broadcast = false` 时，未被归约结果覆盖的位置保留 `trs2z` 原值
- `mask` 参数类型为 `uint8_t`
- `mask = 0` 时不使用掩码
- `mask = 1/2/3` 时，只把对应 `TMASKx` 为真的元素纳入归约集合

## 2. 最小值与最大值归约

### `tredmin` / `tredmax`

- 对整个 `32 x 32` tile 做一次全局归约
- 若 `broadcast = true`，结果写满整个输出 tile
- 否则仅写到 `tile2[row][col]`

### `tredminRow` / `tredmaxRow`

- 每一行独立归约一次
- 若 `broadcast = true`，每行结果广播到该行全部列
- 否则只写到该行的 `col` 位置

### `tredminCol` / `tredmaxCol`

- 每一列独立归约一次
- 若 `broadcast = true`，每列结果广播到该列全部行
- 否则只写到该列的 `row` 位置

## 3. 求和归约

### `tredsum` / `tredsumRow` / `tredsumCol`

求和归约的输出类型不是任意的，而是由输入类型决定：

- 有符号整数输入 -> `SINT32`
- 无符号整数输入 -> `UINT32`
- 浮点输入 -> `F32`

当前实现使用高精度内部累加，再在输出时做类型转换：

- 浮点输出使用 `RNE`
- 整数输出使用带饱和的转换

## 4. 平均值归约

### `tredavg` / `tredavgRow` / `tredavgCol`

- 输出类型与输入类型相同
- 先求和，再除以参与归约的元素个数

当前实现细节：

- 整数平均值采用整数除法，等价于朝 0 方向截断
- 最终结果会裁到目标整数类型可表示范围
- 浮点平均值按目标浮点格式用 `RNE` 舍入

## 5. 2x2 池化

### `tpol2max`

- 对输入 tile 按 `2x2` 窗口做最大池化
- 总共得到 `16 x 16` 个结果
- 结果写入由 `subtile_row/subtile_col` 指定的象限

### `tpol2avg`

- 对输入 tile 按 `2x2` 窗口做平均池化
- 同样得到 `16 x 16` 个结果
- 写回位置规则与 `tpol2max` 相同

象限索引规则：

- `window = 2`
- `subtile_row` 范围 `[0, 1]`
- `subtile_col` 范围 `[0, 1]`
- 结果写入大小为 `16 x 16` 的子区域

## 6. 掩码补充

归约类指令在 `mask = 1/2/3` 时：

- 只把掩码为真的元素加入归约输入集合
- 对全局/行/列归约都是这样
- 对池化也是只把窗口中掩码为真的元素纳入计算
