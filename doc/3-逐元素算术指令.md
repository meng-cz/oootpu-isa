# 3. 逐元素算术指令

本组包含以下指令：

- `tadd`
- `tsub`
- `tmul`
- `tmax`
- `tmin`
- `tand`
- `tor`
- `txor`
- `tsll`
- `tsrl`
- `tsra`
- `tmerge`
- `tnot`
- `tabs`
- `tneg`
- `tsqrt`
- `trsqrt`
- `texp`
- `trcp`
- `trelu`
- `trelu6`
- `tsigmoid`
- `ttanh`
- `tgelu`
- `tcvt`

## 1. 通用规则

除特别说明外：

- 运算都是逐元素进行
- 输出 tile 与输入 tile 的形状相同，始终为 `32 x 32`
- `trdt` 通常允许普通寄存器或 `Temp`
- `trs1zt` 通常允许普通寄存器、`0` 寄存器或 `Temp`
- `trs2z` 通常允许普通寄存器或 `0` 寄存器

## 2. 基本算术

### `tadd` / `tsub`

- 输入输出类型必须完全相同
- 整数按该位宽直接回绕
- 浮点按对应格式做加减，当前实现使用 `RNE`

### `tmul`

- 两个输入类型必须相同
- 输出类型可以与输入相同，或扩成两倍宽度

当前实现支持的扩宽组合：

- `SINT8 -> SINT16`
- `UINT8 -> UINT16`
- `SINT16 -> SINT32`
- `UINT16 -> UINT32`
- `F8E4M3 -> F16`
- `F8E5M2 -> F16`
- `F16 -> F32`

若不扩宽：

- 整数结果按目标位宽截断
- 浮点结果按目标格式舍入为 `RNE`

### `tmax` / `tmin`

- 输入输出类型必须相同
- 整数按有符号或无符号语义比较
- 浮点按对应浮点比较规则比较

## 3. 位运算与移位

### `tand` / `tor` / `txor` / `tnot`

- 输入输出类型必须相同
- 仅使用类型位宽，不关心有符号/无符号/浮点编码含义

### `tsll` / `tsrl` / `tsra`

- `trdt` 与 `trs1zt` 类型必须相同
- 左移、逻辑右移、算术右移
- 位移量来自 `trs2z` 的对应元素
- `tsra` 在有符号整数上会保留符号位
- 仅使用类型位宽，不关心有符号/无符号/浮点编码含义

## 4. 选择与一元变换

### `tmerge`

- 输入输出类型必须相同
- 当前 `funcsim` 读取隐式 `TMASK1`
- 若 `TMASK1[i][j]` 为真，结果取 `trs1zt[i][j]`
- 否则取 `trs2z[i][j]`

### `tabs`

- 输入输出类型必须相同
- 逐元素取绝对值

### `tneg`

- 输入输出类型必须相同
- 逐元素取相反数

## 5. 近似数学函数

以下指令在头文件中标记为“非精确指令”，`funcsim` 也调用了近似实现：

- `tsqrt`
- `trsqrt`
- `texp`
- `trcp`
- `tsigmoid`
- `ttanh`
- `tgelu`

这些指令：

- 只接受浮点类型
- 输入输出类型必须相同
- 支持 `F8E4M3 / F8E5M2 / F16 / F32`

## 6. 激活函数

### `trelu`

- 逐元素执行 `max(x, 0)`

### `trelu6`

- 逐元素执行 `min(max(x, 0), 6)`
- 对浮点类型，常数 `6` 会按目标浮点格式编码

## 7. 类型转换 `tcvt`

```cpp
void tcvt(const TOPRand &trdt, const TOPRand &trs1t, TRM rm, TOV ov, uint32_t max = 0, uint32_t min = 0);
```

语义：

- 先把输入元素提升为内部精确表示
- 再按目标类型执行舍入和转换
- 最后根据 `ov` 决定默认、饱和或裁剪行为

控制项：

- `rm`：`RNE / RTZ / RDN / RUP`
- `ov = DEF`：默认溢出行为
- `ov = SAT`：饱和到目标类型范围
- `ov = CLIP`：先饱和转换，再裁剪到 `[min, max]`

补充说明：

- `funcsim` 中 `SAT` 和 `CLIP` 都会启用“饱和式转换”
- 当 `ov = CLIP` 时，整数比较按目标整数类型解释 `min` 和 `max`，浮点比较按目标浮点编码值解释
