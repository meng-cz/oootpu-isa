# OOOTPU 基础定义

本文档整理 `isa/oootpu.hpp` 中的数据类型、寄存器和操作数约定，并补充 `funcsim` 中可以直接观察到的行为。

## 1. Tile 与寄存器模型

- `TILE_SIZE = 32`
- 每个 Tensor tile 的逻辑形状固定为 `32 x 32`
- 每个元素的逻辑数据类型由 `TOPRand::dtype` 指定

其中：

- 普通 Tensor 寄存器数量由 `getTRegNum()` 返回，当前模拟器实现为 `512`
- 普通寄存器物理上按 8 bit bank 组织
- 16 bit 元素会占用连续 2 个寄存器槽位
- 32 bit 元素会占用连续 4 个寄存器槽位
- `Temp` 寄存器是一个隐式单 tile 临时寄存器
- `Mask` 寄存器有 3 个隐式单 tile 布尔寄存器：`TMASK1/TMASK2/TMASK3`
- `Zero` 寄存器是只读全 0 虚拟寄存器

## 2. 数据类型 `TDType`

支持的数据类型如下：

| 类型 | 位宽 | 类别 | 用于GEMM输入 | 用于GEMM输出 | 用于算术单元 |
| --- | --- | --- | --- | --- | --- |
| `SINT8` | 8 | 有符号整数 | ✅ | ❌ | ✅ |
| `UINT8` | 8 | 无符号整数 | ✅ | ❌ | ✅ |
| `F8E4M3` | 8 | 8 位浮点，4 位指数、3 位尾数 | ✅ | ❌ | ✅ |
| `F8E5M2` | 8 | 8 位浮点，5 位指数、2 位尾数 | ✅ | ❌ | ✅ |
| `SINT16` | 16 | 有符号整数 | ✅ | ✅ | ✅ |
| `UINT16` | 16 | 无符号整数 | ✅ | ✅ | ✅ |
| `F16` | 16 | 半精度浮点 | ✅ | ✅ | ✅ |
| `SINT32` | 32 | 有符号整数 | ❌ | ✅ | ✅ |
| `UINT32` | 32 | 无符号整数 | ❌ | ✅ | ✅ |
| `F32` | 32 | 单精度浮点 | ❌ | ✅ | ✅ |

## 3. 操作数 `TOPRand`

Tensor 操作数定义如下：

```cpp
struct TOPRand {
    TRegIdx index;
    TDType dtype;
    TOPRandType type;
};
```

字段含义：

- `index`：普通寄存器编号
- `dtype`：当前操作数的元素类型
- `type`：寄存器类别，取值为 `NORM/TEMP/ZERO/M1/M2/M3`

几个预定义特殊操作数：

- `TTEMP`：Temp 寄存器
- `TMASK1/TMASK2/TMASK3`：3 个 Mask 寄存器
- `TZERO`：Zero 寄存器

类型包装辅助函数：

- `S8/U8/F8E4M3/F8E5M2/S16/U16/F16/S32/U32/F32(op)`

它们只修改 `dtype`，不改变操作数的寄存器类别。

使用示例：

```cpp
auto trd = F16(0); // 引用第 0 号普通寄存器（不是 '0' 寄存器），元素类型为 F16
auto trs1 = S16(TTEMP); // 引用 Temp 寄存器，元素类型为 SINT16
auto trs2 = U8(TMASK2); // 引用第 2 个 Mask 寄存器，元素解释为 UINT8 类型
auto trs3 = U32(TZERO); // 引用 '0' 寄存器，元素解释为 UINT32 类型

TRegIdx idx = _allocTReg();
auto trs4 = S32(idx); // 动态指定普通寄存器编号，元素类型为 SINT32
```

## 4. 寄存器类别约束

头文件注释里的“仅普通寄存器 / 允许 0 寄存器 / 允许 Temp 寄存器 / 允许 Mask 寄存器”等描述是指该指令对操作数寄存器类别的约束。

常见类别如下：

- 普通寄存器：真实 Tensor 寄存器文件中的一个逻辑寄存器
- `0` 寄存器：读取时得到全 0 tile，写入无效果
- `Temp` 寄存器：单个隐式临时 tile，可作为部分指令的输入或输出
- `Mask` 寄存器：3 个布尔 tile，元素语义为真或假

一般指令会对操作数寄存器类别有特定要求，例如：
1. 所有指令的显式操作数均能接收普通寄存器。
2. 所有指令的输出操作数不能是 `0` 寄存器，绝大多数指令的输入操作数都可以是 `0` 寄存器。例外如tgather的偏移量输入、tcvt的输入等明确有数值语义的输入不能是 `0` 寄存器。
3. 所有逐元素算术指令、Mask 算术与比较指令、taccget 指令的第一个输入操作数和输出操作数可以是 Temp 寄存器。
4. 所有 Mask 算术指令的操作数、Mask 比较指令的输出操作数可以是 Mask 寄存器。

需要注意：

- `Mask` 在被当成操作数读取时会被展开为8位宽数值 tile，真为 `1`，假为 `0`
- 写入 `Mask` 时，任意非零值都会被视为真

## 5. 内存地址与对齐

访存相关指令统一要求：

- 地址按 `元素字节数 x TILE_SIZE` 对齐
- 也就是必须按“一整行 tile 数据大小”对齐

例如：

- 8 bit 元素：按 `32` 字节对齐
- 16 bit 元素：按 `64` 字节对齐
- 32 bit 元素：按 `128` 字节对齐

## 6. 掩码的一般约定

不同指令使用掩码时，掩码的作用范围不完全相同，具体以各分组文档为准。当前实现中常见模式有：

- `mask` 参数类型为 `uint8_t`
- `mask = 0`：不使用掩码
- `mask = 1/2/3`：分别使用 `TMASK1/TMASK2/TMASK3`

- 逐元素访存：按元素决定是否读写
- `gather/scatter`：按 `rowsel` 指定那一行的掩码，决定每一整行是否参与
- `tmerge`：掩码为真时选择 `trs1`，否则选择 `trs2`
- 归约：只把掩码为真的元素纳入归约集合

## 7. 舍入与溢出枚举

### 7.1 舍入模式 `TRM`

- `RNE`：Round to Nearest, ties to Even
- `RTZ`：Round towards Zero
- `RDN`：Round Down
- `RUP`：Round Up

### 7.2 溢出行为 `TOV`

- `DEF`：默认行为
- `SAT`：饱和到目标类型可表示范围
- `CLIP`：裁剪到调用者给定的 `[min, max]`

`tcvt` 和 `taccget` 会直接使用这些控制位。
